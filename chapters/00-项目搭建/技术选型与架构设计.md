### 依赖规则

```
products → features → capabilities → commons
                  ↘ commons ↗

features 之间不能互相依赖
capabilities 之间不能互相依赖
所有层都可以依赖 commons
```

## 三、关键架构决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| UIAbility 数量 | 1 个主 UIAbility | 避免状态碎片化，统一初始化和播放器/下载/会话协调 |
| 导航 | Navigation + NavPathStack + 类型化路由 | 集中管理，支持深链/登录拦截/WebView 路由 |
| 状态管理 | AppStorage 放轻量标志（token/isVip/未读数）；核心状态用 Store/Service | 避免 AppStorage 成为全局垃圾桶 |
| 数据层 | Repository 模式（RemoteDataSource + LocalDataSource） | 支持离线场景（历史/下载/草稿），可维护 |
| 播放器 | IPlayerEngine 抽象层 → 统接火山引擎 TTSDK | 保持端侧一致性，完整支持视频解密与播放调度 |
| 图片加载 | ImageKnife 封装在 capabilities/image | 统一策略（占位图/错误图/缓存），可替换 |
| 推送 | 华为 Push Kit | 鸿蒙原生生态，减少依赖层 |
| 模块打包 | 全部 HAR（静态共享包） | 编译时集成，简单可靠 |

## 六、权益逻辑（1:1 还原 Android）

### VIP 状态校验

```
isVip() → 读 Preferences 中后端返回的布尔值
写入时机：登录时（loginBean.vip）、支付轮询时（PayResultPoller）
不做本地过期时间判断，不做定时刷新
```

### 试看限制

```
电影：360 秒（6 分钟）— 硬编码
剧集：180 秒（3 分钟）— 硬编码
判断：if (isVip || isFree) return false，否则到时间暂停 + 弹 VIP 购买窗
```

### 缓存播放权限

```
播放前校验：verifyCachePlayPermission()
  if (priceType != "0" && isPurchased != 1 && !isVip) → 弹窗拦截"VIP已过期"
通过校验后：isFree=true，跳过试看限制
setCachePlayMode(true) → VipLayer.show() 被阻止
```

### 投屏权限

```
还原 LeboPushActivity 逻辑：
  if (isFree() || isPurchased || isVip()) → 允许投屏
  否则 → 弹 VIP 购买窗
```

## 七、播放器架构

### 两种播放模式

| | 影视详情播放 (SingleContentPlayerSession) | 短视频 Feed 播放 (FeedPlayerCoordinator) |
|---|---|---|
| 播放器实例 | 单个，长生命周期 | 单个，快速切换 |
| 控制 UI | 完整（进度条/全屏/倍速/弹幕/选集） | 极简（暂停/进度指示） |
| 预加载 | 当前集 | 上下各 1-2 条 |
| 生命周期 | 页面级 | 列表项可见性驱动 |
| 全屏 | 横屏 | 竖屏 |
| VIP 限制 | checkVipLimit() 试看倒计时 | priceType=="2" 单集解锁 |

### 抽象层接口

```
capabilities/playback/
├── IPlayerEngine          # play/pause/seek/setSpeed/setSource/release
├── IPlayerController      # UI 控制层调用
├── IPlaybackSession       # 会话管理（当前源/进度/状态）
├── IPlayerEventListener   # 状态/进度/错误/首帧/缓冲回调
├── MediaSource            # 播放源模型（url/cacheKey/headers）
├── PlaybackConfig         # 配置（倍速/清晰度/起始位置）
├── PlaybackState          # 状态枚举（Idle/Preparing/Playing/Paused/Completed/Error）
├── VolcPlayerEngine       # 火山引擎 SDK 实现
└── PlaybackAnalyticsAdapter  # 埋点（首帧/缓冲/完播/退出原因）
```

## 八、路由设计

### 统一 URL 契约

所有入口（推送/分享/WebView/深链）走同一个解析器：

```
rrys://video/detail?id={movieId}
rrys://video/short?id={videoId}
rrys://user/profile?id={userId}
rrys://search?keyword={keyword}
rrys://vip/product
rrys://message
rrys://web?url={encodedUrl}&title={title}
rrys://nft/store
```

### 路由拦截

```
登录拦截：需要登录的页面（发布动态/评论/购买/个人中心编辑等）
  → 未登录时跳转登录页，登录成功后回到原目标页
```

## 九、本地存储策略

### RDB（关系型数据库）

| 表 | 用途 |
|----|------|
| play_history | 播放历史（movieId/episodeId/progress/timestamp） |
| search_history | 搜索历史（keyword/timestamp） |
| download_meta | 下载元数据（taskId/movieId/episodeId/url/quality/progress/status/localPath/priceType/isPurchased） |

### Preferences（轻量存储）

| Key | 用途 |
|-----|------|
| token | 登录 Token |
| userId | 用户 ID |
| isVip | VIP 状态（后端布尔值） |
| vipExpiredTime | VIP 过期时间（仅展示用） |
| nick / headImg / gender | 用户基本信息 |
| bitratePreference | 清晰度偏好 |
| notificationEnabled | 通知开关 |

### 不做本地缓存 schema 的数据

影视列表、评论、消息、动态等业务数据以后端为准，不做本地 ORM 映射。
