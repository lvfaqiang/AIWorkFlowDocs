# 模块实现方案模板

> 适用于：模块开发工作流 ① 生成模块实现方案。
> 用途：为 Coding Agent 提供"完全预消化"的实现上下文，零探索成本。
> 存放位置：`chapters/{NN}-{模块名}/impl-plan.md`
> 前置条件：该模块的 `ui-plan.md` 已完成。

---

## 六、Todo 列表

> 编号规则：T-{模块编号}{两位序号}，如 T-0201 表示模块02的第1个Todo。
> 每个 Todo 完成后勾选 checkbox 并 git commit。

- [ ] T-{NNXX}: {Todo 标题}
  - 涉及文件：{文件路径}
  - 验收标准：{具体的可验证条件}

- [ ] T-{NNXX}: {Todo 标题}

- [ ] T-{NNXX}: 构建验证
  - 验收标准：执行构建命令，零错误零警告

### Todo 拆分原则

- 每个 Todo 是一个原子操作，完成后项目可构建
- 按依赖顺序排列（模型 → 数据层 → 逻辑层 → UI 层）
- 粒度适中：太粗（>2小时）需拆分，太细（<15分钟）可合并
- 最后一个 Todo 固定为"构建验证"
